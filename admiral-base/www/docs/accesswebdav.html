ADMIRAL data file web file system access (WebDAV)

Selected ADMIRAL data files may be accessed using the web file access protocol, using one of the following forms of URI: 

	http://%{HOSTFULLNAME}/data/private/username
	http://%{HOSTFULLNAME}/data/shared/username
	http://%{HOSTFULLNAME}/data/collab/username
A username and password will be required. Access controls are enforced as described for local network file access page.

Web file access (WebDAV) support is slower and sometimes less reliable than local network file sharing. It is provided for two main reasons:

  1. It can be used from a system outside the Zoology Department network.
  2. It can be used by web browser based applications used for annotation, visualization and other tasks.

Web file access using Windows

Currently, the Windows web file access software does not work with the ADMIRAL server and file protection arrangements — we'll be looking in to this.

Web file access using MacOS

To connect a web file system share from MacOS, use the Finder "Go > Connect to server..." menu command (or COMMAND-K shortcut), and enter http://%{HOSTFULLNAME}/data/private/username, or one of the other forms shown above, as the server address. The system will then prompt for a username and password.

Web file access using Linux

Under Linux with GNOME, the ADMIRAL web file system content can be accessed in similar fashion to a local file system, but using the web address:

Use the "Places > Connect to server ..." menu command. In the popup window, from the "Service type" drop-down menu, select "WebDAV (HTTP)". Under "Server" enter %{HOSTFULLNAME} and under "Folder" enter data/username, data/shared/username or data/collab/username. Under "Username" enter username and then click on "Connect". Enter the specified user's password into the next popup window to appear.

An alternative mechanism for reading and writing files under Linux is to use the cadaver utility, specifying one of the URI forms listed above.
